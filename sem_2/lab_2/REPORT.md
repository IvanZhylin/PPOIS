# Отчёт по разработанной системе

## 1. Назначение

Разработана информационная система для работы с записями о железнодорожных рейсах (вариант 13):
- номер поезда;
- станция отправления;
- станция прибытия;
- дата/время отправления;
- дата/время прибытия;
- время в пути (вычисляется автоматически).

## 2. Архитектура

Система реализована в соответствии с MVC и SOLID.

- `Model`:
  - `TrainRecord` - доменная сущность;
  - `SearchCriteria` - модель условий поиска/удаления;
  - `Page` - модель пагинации.
- `View`:
  - главное окно (`MainWindow`);
  - дочерние диалоги: добавление, поиск, удаление.
- `Controller`:
  - `MainController` координирует UI и бизнес-логику.
- `Service`:
  - `TrainService` инкапсулирует операции приложения (поиск, удаление, импорт/экспорт, пагинация).
- `Repository`:
  - `ITrainRepository` (абстракция);
  - `SqliteTrainRepository` (конкретная реализация SQLite).
- Хранение XML:
  - `IXmlStorage` (абстракция);
  - `DomSaxXmlStorage` (DOM-запись, SAX-чтение).

### SOLID

- **S**: классы разделены по ответственности (UI, хранение, бизнес-логика).
- **O**: расширение возможно через новые реализации интерфейсов (`ITrainRepository`, `IXmlStorage`) без изменения контроллера.
- **L**: контроллер/сервис работают через абстракции, не зависят от деталей реализации.
- **I**: интерфейсы узкие и предметные.
- **D**: высокоуровневые компоненты зависят от абстракций, внедрение через конструкторы.

## 3. Реализованные требования

1. Главное окно + дочерние диалоги реализованы.
2. Вызов через меню, команды продублированы на toolbar.
3. Формирование массива через диалог добавления.
4. Поиск в отдельном диалоге по условиям варианта.
5. Удаление в отдельном диалоге по условиям варианта.
6. Пользователю выводится факт удаления и количество удалённых записей.
7. Отображение текущего массива в главном окне.
8. Хранение записей в базе данных SQLite.
9. Альтернативный вывод массива в виде дерева.
10. Сохранение/загрузка в XML через стандартные диалоги.
11. Сохранение/загрузка в/из выбранной SQLite БД через стандартные диалоги.
12. XML: запись DOM, чтение SAX.
13. Используются корректные типы данных (`datetime`, целые минуты).
14. Постраничный вывод в главном окне и окне поиска:
    - первая/предыдущая/следующая/последняя страницы;
    - смена размера страницы;
    - отображение текущего количества, общего количества, номера текущей и общего числа страниц.

## 4. Сценарии демонстрации системы

### Сценарий 1. Базовая работа с записями
1. Запустить приложение.
2. Добавить несколько записей через диалог "Добавить запись".
3. Убедиться, что записи появились в главной таблице, а "Время в пути" рассчитано автоматически.

### Сценарий 2. Поиск
1. Открыть диалог "Поиск".
2. Задать одно или несколько условий:
   - номер поезда;
   - дата отправления;
   - диапазон времени отправления или прибытия;
   - станция отправления или прибытия;
   - диапазон времени в пути (в минутах).
3. Нажать "Найти", проверить результат и работу пагинации.

### Сценарий 3. Удаление
1. Открыть диалог "Удаление".
2. Задать условия удаления.
3. Подтвердить удаление.
4. Проверить системное сообщение о количестве удалённых записей или отсутствии совпадений.

### Сценарий 4. XML
1. Сохранить текущий массив в XML.
2. Очистить/изменить набор записей.
3. Загрузить данные обратно из XML и проверить восстановление.

### Сценарий 5. Работа с БД-файлами
1. Сохранить текущий массив в выбранный файл SQLite.
2. Подключить другой файл БД через "Загрузить из БД".
3. Убедиться, что отображается другой набор записей.

### Сценарий 6. Таблица и дерево
1. В главном окне переключить режим "Таблица/Дерево".
2. Проверить, что в дереве листовые элементы соответствуют полям записи.

### Сценарий 7. Демонстрационные наборы по 50 записей
1. Выполнить `python3 generate_demo_files.py`.
2. Получить несколько XML-файлов (`demo_files/*.xml`) по 50 уникальных осмысленных записей.
3. Поочередно загружать файлы в приложение для демонстрации.
